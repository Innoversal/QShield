# Security Policy

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a security vulnerability in QShield, please disclose it responsibly:

**Email:** security@qshield.io *(to be configured)*

Please include:
- A description of the vulnerability
- Steps to reproduce
- Potential impact assessment
- Suggested fix if available

We will respond within **48 hours** and work with you on a coordinated disclosure timeline.

## Scope

Security issues in scope:
- Cryptographic implementation flaws
- Protocol design weaknesses
- Timing side-channels in cryptographic code
- CA infrastructure vulnerabilities
- Authentication bypasses

## Out of Scope

- Issues in third-party dependencies (report to them directly)
- Theoretical attacks requiring quantum computers to exploit
- Social engineering attacks

## Disclosure Policy

We follow a 90-day disclosure timeline. We will credit researchers who report valid vulnerabilities unless they prefer to remain anonymous.
